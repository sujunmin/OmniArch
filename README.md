# OmniArch: The Cloud Agent

## How we used OpenAI Technologies

In this hackathon, OmniArch heavily relies on the latest OpenAI models to achieve its core functionalities:

* **GPT-5.6 for Advanced Reasoning:** We utilized GPT-5.6 as the core cognitive engine. Its enhanced logical reasoning capabilities allowed the Agent to mathematically deduce complex infrastructure volumetrics (e.g., calculating GKE node sizing and Cloud SQL database requirements based on target ARR) rather than simply guessing. It also powers the nuanced "translation" of strict technical protocols (like OIDC constraints) into executive-friendly business pitches.
* **Codex for SOW Automation & Integration:** We leveraged Codex to parse, structure, and generate technical documentation. Codex was instrumental in extracting unstructured requirements from rejected SOWs and automatically formatting them into rigid, compliance-ready Fair Market Value (FMV) JSON structures and technical markdown tables.
